### 1. AMD Advancing AI 2026: EPYC Venice on 2nm, MI400 Lineup, and Helios Rack Unveiled
**Source:** [WCCFTech](https://wccftech.com/watch-amd-advancing-ai-2026-event-live-here/) · [TechTimes](https://www.techtimes.com/articles/321257/20260722/amd-advancing-ai-2026-opens-zen-6-venice-helios-open-ai-rack-bet.htm) · [DataCenterDynamics](https://www.datacenterdynamics.com/en/news/amd-unveils-full-mi400-product-lineup-claims-mi500-chips-will-deliver-1000x-increase-in-ai-performance/) · [TechInsider](https://tech-insider.org/amd-advancing-ai-2026/)

At today's Advancing AI 2026 keynote in San Francisco, CEO Dr. Lisa Su unveiled three major product tiers. EPYC "Venice" becomes the first x86 server CPU in volume production on TSMC's 2nm node, packing up to 256 Zen 6 cores (33% more than the current Turin lineup) with AMD claiming >70% performance-per-watt gains. The full MI400 accelerator lineup was specified alongside a 2027 MI500 roadmap preview that claims a 1,000× performance jump over MI300X. The Helios rack-scale platform — targeting Q3 2026 general availability at $5.25M per rack — houses 72 MI455X accelerators and 31TB of HBM4 for up to 3 AI exaflops in a single rack. OpenAI and Meta together have committed 12 gigawatts of AMD accelerator capacity; Microsoft Azure and Oracle are confirmed early Helios customers.

**Why it matters:** AMD is mounting a credible challenge to NVIDIA's data-center monopoly at every layer of the stack simultaneously — silicon, memory, and rack — and the 2nm EPYC launch gives cloud providers a concrete reason to diversify supply chains before NVIDIA's next-gen Rubin chips arrive.

## New Framework

### 2. MCP 2026-07-28 Specification Reaches Release Candidate — Stateless Core, Ships in 5 Days
**Source:** [MCP Blog](https://blog.modelcontextprotocol.io/posts/2026-07-28-release-candidate/) · [WorkOS](https://workos.com/blog/mcp-2026-spec-agent-authentication) · [Stacktree](https://stacktr.ee/blog/mcp-2026-spec-changes) · [AAIF](https://aaif.io/blog/mcp-2026-07-28-whats-changing-and-how-to-migrate) · [SecurityWeek](https://www.securityweek.com/new-enterprise-ready-mcp-specification-brings-new-security-challenges/)

The final Model Context Protocol spec ships July 28 and is in release-candidate review now. Lead maintainers are calling it the largest revision since launch: the protocol core becomes fully stateless (sessions and the initialization handshake are removed), the authorization layer is rewritten, a new Tasks extension supports long-running agentic work, and an MCP Apps extension enables server-rendered UIs. A stateless MCP server can now run behind a plain round-robin load balancer instead of requiring sticky sessions and a shared session store. SecurityWeek notes the new spec also resets the attack-surface conversation — the authentication overhaul and stateless design introduce new trust-boundary questions that security teams will need to evaluate.

**Why it matters:** Stateless MCP dramatically lowers the operational cost of hosting remote tools at scale, which should accelerate enterprise adoption — but every existing SDK and server that manages session state needs to migrate before the July 28 cutover.

### 3. Vercel AI SDK 7 — Durable WorkflowAgent for Production
**Source:** [Vercel Changelog](https://vercel.com/changelog/ai-sdk-7) · [Developers Digest](https://www.developersdigest.tech/blog/vercel-ai-sdk-7-production-agents) · [AlphaSignal](https://alphasignal.ai/news/vercel-s-ai-sdk-7-turns-fragile-prototype-agents-into-production-ready-systems)

Vercel shipped AI SDK 7 on June 25 with three agent abstractions: ToolLoopAgent (in-memory, prototype-grade), WorkflowAgent (durable, stateful — survives deploys by persisting tool state and supporting step-level retries and human-in-the-loop approval), and a new top-level reasoning mode. Additional additions include overhauled OpenTelemetry telemetry, provider-independent file uploads, typed tool context, and granular timeout controls.

**Why it matters:** WorkflowAgent closes the gap between demos and production — agents that previously lost state on redeploy or transient failure can now run multi-hour workflows reliably, which is a prerequisite for the long-horizon agentic pipelines most teams are trying to build.

## New Agent Tool

### 4. Alterion Launches Draco — Runtime Control Plane for Enterprise AI Agents
**Source:** [PR Newswire](https://www.prnewswire.com/news-releases/alterion-launches-draco-a-runtime-control-plane-for-enterprise-ai-agents-302827818.html) · [Alterion](https://www.alterion.ai/platform/draco) · [Dealroom](https://app.dealroom.co/news/feed/alterion-launches-draco-runtime-control-platform-for-enterprise-ai-agents)

Alterion made Draco generally available on July 16. Draco sits between AI agents and enterprise infrastructure as a dedicated runtime control layer, using its Helix intelligence engine to observe every prompt, action, and payload in real time, model agent intent, and enforce programmable guardrails before high-risk actions (data deletion, production changes) execute — with no agent code changes required. The platform covers all OWASP Top 10 for Agentic Applications and maps to SOC 2, ISO 42001, and EU AI Act requirements. Target buyers are organizations in regulated industries deploying agents across multiple clouds and vendors.

**Why it matters:** Draco's code-free integration model lowers the barrier for enterprises already running production agents to add compliance-grade governance retroactively — directly addressing the enforcement gap that Gartner and CISA flagged this month as the top risk in autonomous agent deployments.

### 5. Frigade Launches Skills — No-Code In-App AI Agent That Acts on Behalf of Users
**Source:** [PR Newswire](https://www.prnewswire.com/news-releases/frigade-launches-skills-giving-any-product-an-ai-assistant-that-takes-action-for-users-no-code-required-302824744.html) · [Y Combinator Launch](https://www.ycombinator.com/launches/RLV-frigade-skills-the-in-app-ai-agent-that-learns-your-product-and-acts-for-your-users) · [Yahoo Finance](https://finance.yahoo.com/technology/ai/articles/frigade-launches-skills-giving-product-120000158.html)

Frigade (YC-backed) launched Skills on July 14, extending its user-onboarding platform into agentic territory. Skills trains on a SaaS product by using it autonomously, documents the actions it discovers, then surfaces those actions for team approval via a dashboard — no integration code required. End users can ask in plain language for any in-product task and the assistant executes it using the product's live UI; early customers in logistics, automotive, developer tools, and HR report significant support-ticket deflection.

**Why it matters:** Skills reduces the engineering cost of adding an in-product agent from weeks of custom integration to a dashboard approval step, making agentic UX accessible to product teams that can't dedicate engineering bandwidth to it — a meaningful unlock for the long tail of B2B SaaS.

## Research

### 6. Agent4cs Accepted at EUMAS 2026 — Multi-Agent Code Summarization for Large Codebases
**Source:** [arXiv — cs.AI July 2026](https://arxiv.org/list/cs.AI/current)

Agent4cs, a multi-agent system for code summarization in large hierarchical codebases, was accepted to the main track of the 23rd European Conference on Multi-Agent Systems (EUMAS 2026). The system decomposes a repository into subgraphs and assigns specialized agents to each, then synthesizes summaries bottom-up — outperforming single-agent and retrieval-augmented baselines on recall and coherence at scale.

**Why it matters:** Code summarization is a high-value developer-productivity task where single-agent approaches hit context-window limits on real enterprise codebases; Agent4cs's hierarchical decomposition pattern is directly applicable to other large-structure understanding problems like architecture review and dependency auditing.
