## New Agent Tool

### 1. OpenAI Responses API Ships Programmatic Tool Calling and Multi-Agent Beta
**Source:** [OpenAI](https://openai.com/index/gpt-5-6/) · [MarkTechPost](https://www.marktechpost.com/2026/07/09/openai-releases-gpt-5-6-a-three-tier-model-family-with-programmatic-tool-calling/) · [ChatForest](https://chatforest.com/builders-log/openai-gpt-5-6-sol-ga-july-9-api-surface-programmatic-tool-calling-builder-guide/) · [AI Joao Queiros](https://www.ai.joaoqueiros.com/blog/gpt-5-6-codex-programmatic-tools-subagents-work-os)

With the general availability of GPT-5.6 this week, OpenAI shipped two new API capabilities in the Responses API: **Programmatic Tool Calling** and a **Multi-Agent beta**. Programmatic Tool Calling lets GPT-5.6 write and execute model-authored JavaScript in an isolated V8 runtime — with no network access — that coordinates tool calls, filters large intermediate outputs, and adapts its workflow step-by-step, all within a single Zero Data Retention–compatible request. The Multi-Agent beta (also in the Responses API) allows a single API request to spin up concurrent GPT-5.6 subagents and synthesize their results before returning. GPT-5.6 Sol pricing is $5/$30 per 1M input/output tokens; Terra is $2.50/$15; Luna is $1/$6.

**Why it matters:** These two features together push the tool-calling bottleneck almost entirely into the model — fewer round trips, fewer tokens, and multi-threaded agent work all within a single API call. For production agentic applications, Programmatic Tool Calling is a meaningful step toward self-contained task execution that doesn't require developer orchestration of every step.

## New Framework

### 2. Microsoft Agent Framework Ships 1.11.0 (Python) and 1.13.0 (.NET) — MCP Tool Discovery, Skills, AG-UI
**Source:** [GitHub Releases](https://github.com/microsoft/agent-framework/releases) · [Microsoft Dev Blog](https://devblogs.microsoft.com/agent-framework/) · [Releasebot](https://releasebot.io/updates/microsoft)

Microsoft released **Agent Framework 1.11.0** (Python) and **1.13.0** (.NET) in near-simultaneous updates this week. The Python 1.11.0 release adds smarter agent sessions, **progressive MCP tool discovery** (tools surface to the agent incrementally as they become available rather than requiring a full manifest upfront), new skill primitives and hosting helpers, and improved AG-UI protocol support with stronger provider coverage. The .NET 1.13.0 release adds expanded skills APIs, new built-in file editing tools, richer approval and caching options, and Foundry Hosting improvements for session isolation and local development, alongside fixes for workflow and dependency issues with several breaking changes flagged.

**Why it matters:** Progressive MCP tool discovery is the standout change: it lets agents start working before all tool schemas are loaded, reducing startup latency in tool-heavy deployments and unblocking integrations with slow or large MCP servers. The parallel Python/dotnet cadence signals that Microsoft is actively maintaining parity across both ecosystems — important for enterprises that run polyglot agentic stacks.

## Research

### 3. arXiv: "Memory in the Loop — In-Process Retrieval as Extended Working Memory for Language Agents"
**Source:** [arXiv 2607.05690](https://arxiv.org/pdf/2607.05690)

A new paper (arXiv:2607.05690, July 2026) proposes treating **retrieval as an in-process operation within the agent's execution loop** rather than as a preprocessing or tool-call step. The authors introduce a framework in which a language agent maintains an "extended working memory" by issuing retrieval operations mid-reasoning, using retrieved context to update its in-context state without returning to the outer orchestration layer. Experiments across multi-hop reasoning and long-context tasks show substantial gains over retrieve-then-read and tool-call-based retrieval baselines.

**Why it matters:** Current RAG architectures mostly treat retrieval as a one-shot pipeline step that happens before the model thinks. In-process retrieval blurs the line between memory and reasoning — bringing agent cognition architecturally closer to how humans access long-term memory during active thought. If the performance gains replicate, this approach could shift how production memory systems are built.

## Research

### 4. mem0 Releases "State of AI Agent Memory 2026" — Episodic and Semantic Memory Still Major Gaps
**Source:** [mem0 Blog](https://mem0.ai/blog/state-of-ai-agent-memory-2026) · [arXiv 2606.24775](https://arxiv.org/pdf/2606.24775) · [arXiv 2602.19320](https://arxiv.org/pdf/2602.19320)

mem0 published a **State of AI Agent Memory 2026** benchmark report alongside a cluster of concurrent arXiv work, including "Are We Ready For An Agent-Native Memory System?" (arXiv:2606.24775) and an anatomy of agentic memory systems (arXiv:2602.19320). The combined finding is that while agent memory has advanced significantly in 2025–26, **episodic memory** (maintaining event sequences across sessions) and **semantic generalization** (extracting reusable knowledge from experience) remain the two biggest architectural gaps. Most current production implementations rely on simple key-value or vector stores that handle neither well at scale.

**Why it matters:** Memory is increasingly seen as the primary determinant of long-horizon agent capability. The report provides a structured taxonomy and empirical baselines that teams can use to evaluate their own memory implementations — useful both for benchmarking existing systems and for prioritizing which memory type to invest in next.

### 7. Crunchbase: Global AI Startup Investment Hits Record $510B in H1 2026 — 88% to U.S. Firms
**Source:** [Crunchbase H1 2026 Report](https://news.crunchbase.com/venture/global-startup-exits-ipo-ma-soar-ai-q2-h1-2026/) · [Crunchbase Q2 2026](https://news.crunchbase.com/venture/na-startup-funding-ma-shattered-records-ai-q2-2026/)

Crunchbase's H1 2026 global startup investment report shows investors poured **$510 billion into startups worldwide in the first half of 2026** — a record — with AI investments dominating. Q2 2026 alone saw more than $205 billion deployed across 5,000+ rounds. Notably, **88% of AI startup funding went to U.S.-based companies**, with OpenAI, xAI, and Anthropic accounting for a disproportionate share of mega-rounds. The report identifies AI infrastructure (compute, inference, tooling), AI health/bio, and autonomous systems as the three leading sub-verticals by capital deployed.

**Why it matters:** $510B in six months is not a bubble — it's a capital reallocation at a pace that matches or exceeds the dot-com era in absolute terms. The 88% U.S. share is strategically significant: it means that almost all frontier-level AI development is funded from one country's capital markets, a concentration that will increasingly drive both geopolitical attention and regulatory pressure. The infrastructure sub-vertical's prominence confirms that the "picks and shovels" layer (compute, serving, observability) is now as investable as application-layer AI.
